# 开发文档

## 1. 开发环境

- Go 1.22+
- Git
- 可访问主从主机的 SSH 网络环境（后续联调使用）

## 2. 目录结构

```text
cmd/replpilot/             # 程序入口
internal/command/          # Cobra 命令定义
internal/config/           # 配置读写
internal/state/            # 任务与检查结果状态
docs/                      # 设计与开发文档
```

## 3. 本地运行

```bash
go mod tidy
go run ./cmd/replpilot --help
```

## 4. 当前命令

- `replpilot preflight --source <id> --replica <id> [--fix]`
- `replpilot init`
- `replpilot host add/list/remove ...`
- `replpilot source add/list/remove ...`
- `replpilot bootstrap --source <id> --replica <id> [--mode auto] [--dry-run]`
- `replpilot status --source <id> [--replica <id>]`
- `replpilot diagnose --source <id> [--replica <id>]`

## 5. 后续开发顺序

1. 实现真实 MySQL 复制执行（`CHANGE REPLICATION SOURCE TO` / `START REPLICA`）
2. 增加配置加密与敏感信息保护
3. 补齐 checkpoint 回滚与失败重试策略
4. 增加端到端联调与跨端打包

## 6. preflight 当前检查项

- source 配置完整性（`master_host/master_port/repl_user`）
- replica 配置完整性（`address/port/user/auth_type/key_path`）
- 远程 SSH 连通（`echo ok`）
- 远程 sudo 能力（`sudo -n true`）
- mysql 客户端存在性（`command -v mysql`）
- mysql 版本可读（`mysql --version`）
- 从库到主库端口连通（优先 `nc`，回退 `/dev/tcp`）

## 7. 状态与诊断

- `bootstrap` 会写入任务状态（`~/.replpilot/tasks.json`）
- `preflight` 会写入最近检查结果（`~/.replpilot/preflight_checks.json`）
- `status` 读取最新任务并展示步骤与摘要
- `diagnose` 汇总 preflight 与任务状态给出建议
